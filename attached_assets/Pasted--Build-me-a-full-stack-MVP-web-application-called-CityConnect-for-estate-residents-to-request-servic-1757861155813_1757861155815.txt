"Build me a full-stack MVP web application called CityConnect for estate residents to request services. Focus on Service Providers (artisans/repairs) and Market Runs (errands/deliveries) as the first features. The application should have the following:"

Tech Stack

Frontend: React (with Vite or Next.js if possible)

Backend: Node.js with Express

Database: MongoDB (use Mongoose for schemas)

Authentication: JWT-based auth with bcrypt password hashing

UI: Clean, mobile-first design (TailwindCSS)

Core Features

User Authentication

Resident signup/login (email, phone number, password)

Service Provider signup/login (artisan, runner for errands)

JWT-based auth with role-based access

Service Categories (Initial)

Artisan Repairs: residents can browse available artisans (electrician, plumber, carpenter).

Market Runs/Errands: residents can request errands (groceries, deliveries, parcel runs).

Request Flow

Resident selects category (Artisan / Market Run).

Fills request form (description, urgency, location, budget).

Request saved in DB with status = pending.

Providers see available requests.

Provider accepts a request → status updates to “assigned”.

Resident gets notified of assignment.

Order Tracking

Resident can view request history and status (pending, assigned, completed).

Provider can view accepted jobs and update status to “in progress” → “completed”.

Payments (basic)

Simulate wallet-based payment (no need for full integration yet).

On completion, resident can mark job “paid”.

Keep placeholder hooks for Paystack/Flutterwave integration.

Admin Dashboard (basic)

View all users (residents + providers).

View all service requests.

Approve/reject service providers.

Database Models

User: { name, email, phone, passwordHash, role: [resident, provider, admin], rating }

ServiceRequest: { category, description, residentId, providerId, status, budget, createdAt }

Wallet (optional): { userId, balance, transactions }

Pages / Screens

Landing Page (welcome, login, register)

Resident Dashboard:

Book Artisan Repair

Request Market Run

Track Orders

Provider Dashboard:

View Available Jobs

Accept/Update Job Status

Admin Dashboard:

Manage Users & Providers

View All Requests

Extra

Use Tailwind for clean mobile-first UI.

Add simple notifications (toast messages).

Code should be modular and scalable for future services (laundry, transport).
